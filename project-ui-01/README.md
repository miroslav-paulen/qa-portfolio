
Pojmy 


Testy :
            
            Funkčné : Testovanie funkcionality podla špecifikácie
            
            Regresné : Testovanie že existujúce funkcie sa nepokazili po oprave, novej funkcii, úprave kódu
            
            Smoke : Overenie že aplikácia je testovatelná


Závažnosť :
           
            Critical - systém padá, nedá sa použiť
            
            High     - Hlavná funkcionalita je rozbitá
            
            Medium   - niečo nefunguje ale existuje workaround
            
            Low      - kozmetické chyby

Priorita :
            
            High   - musí sa opraviť hneď
            
            Medium - opraviť čoskoro
            
            Low    - môže počkať do dalšieho releasu

Životný cyklus defektu :

            1. New         -  Nový nahlásený bug
                              čaká na priradenie vývojárovy
                              
            2. In progress -  Vývojár opravuje bug
                              analýza, oprava, testovanie
                              
            3. Resolved    -  Bug je opravený / odstránený
                              pripravený na testovanie
                              
            4. Retest      -  Tester skontroluje či bol bug naozaj odstránený
                              overý aj okolité scénare (regresne testovanie)
                              
            5. Closed      -  Tester potvrdí že bug bol odstránený
                              Bug sa uzavrie

            Ďalšie možné stavy :  Rejected
                                  Duplicate
                                  Cannot reproduce
                                  Reopened
